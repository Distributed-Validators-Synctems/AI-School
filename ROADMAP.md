# 🗺 План развития ИИ-Школы «Дирижер» (Roadmap)

Этот документ описывает стратегическое видение и этапы превращения интенсивного курса в глобальную образовательную платформу для подготовки Дирижеров ИИ.

---

## 👁 Видение (Vision)
Создать экосистему, где любой человек (независимо от навыков программирования) может стать архитектором сложных IT-продуктов, используя ИИ в качестве своей профессиональной инженерной команды. От ручного обучения с менторами — к полностью автоматизированной платформе под управлением ИИ-агентов.

---

## 🚀 Основные этапы развития

### Фаза 1: Фундамент и Методология (Текущий этап)
*Цель: Подготовить качественный контент и запустить первый поток.*
- [x] Разработка учебной программы на 10 сессий.
- [x] Создание детальных методичек для менторов (Сессии 01–10).
- [x] Подготовка "Чит-листа" полезных ссылок и ресурсов.
- [x] Настройка регистрационных форм и Telegram-сообщества.
- [/] Сбор и запуск пилотной группы студентов.

### Фаза 2: Обкатка и Сбор данных
*Цель: Провести живое обучение и подготовить базу для автоматизации.*
- [ ] Проведение 10 сессий в живом формате (Live).
- [ ] Полная видеозапись всех занятий и разборов.
- [ ] Сбор обратной связи: что было слишком сложно, что — слишком просто.
- [ ] Корректировка программы (Refactoring): удаление лишнего, усиление слабых мест.
- [ ] Формирование базы знаний "вопрос-ответ" (FAQ) на основе реальных проблем студентов.

### Фаза 3: Автоматизация и ИИ-Агенты (Conductor Engine)
*Цель: Передать роль учителя ИИ-агентам.*
- [ ] Обучение специализированных ИИ-агентов на основе видеозаписей и методичек.
- [ ] Разработка системы автоматической проверки домашних заданий агентами.
- [ ] Создание интерактивных "песочниц", где ИИ-агент подсказывает студенту в реальном времени.
- [ ] Запуск потоков, где теорию дает ИИ, а живой ментор подключается только для консультаций.

### Фаза 4: Образовательная платформа (AI-School Web)
*Цель: Запуск платформы в стиле boot.dev, но для управления агентами.*
- [ ] Создание веб-интерфейса платформы с личным кабинетом студента.
- [ ] Геймификация обучения: дерево навыков (Skill Tree), уровни и достижения.
- [ ] Интеграция ИИ-наставника прямо в интерфейс сайта (чат/голос).
- [ ] Разработка системы квестов (как в boot.dev), где нужно не "писать JS", а "управлять флотом агентов" для решения задачи.

### Фаза 5: Масштабирование и Монетизация
*Цель: Финансовая устойчивость и неограниченный рост.*
- [ ] Внедрение подписочной модели (SaaS).
- [ ] Доступ к уникальным MCP-серверам и инструментам по подписке.
- [ ] Создание биржи выпускников-дирижеров для заказчиков.
- [ ] Партнерство с инструментами (Anthropic, RunPod, Loveable) для специальных условий студентам.
