# ИИ-Школа «Дирижер»: Создание IT-продуктов без навыков программирования

- [Для англо-говорящих (For english-speakers)](https://github.com/Distributed-Validators-Synctems/AI-School/blob/master/README.md)

**Интенсивный курс: 20 часов (10 занятий по 2 часа)**

Эта программа создана для тех, кто хочет перестать просто «чатиться» с ИИ и начать использовать его как полноценную команду инженеров. Мы учим **Vibe-coding** — методологии, где вы выступаете в роли архитектора и дирижера, а всю техническую работу выполняют автономные ИИ-агенты.

---

## 🛠 Стек инструментов курса
В процессе обучения мы развернем и настроим «кокпит» управления разработкой:
* **Оркестраторы:** Antigravity, Claude Code (CLI).
* **Среда:** VS Code + Windsurf / Cursor.
* **Оптимизация:** jcodemunch, MCP (Model Context Protocol).
* **Инфраструктура:** RunPod, Vast.ai (аренда GPU), Docker.
* **Интерфейсы:** Bolt.new, Lovable.

---

## Учебный план

### Модуль 1: Фундамент и Инструменты прямого действия

### Занятие 01: Вход в Vibe-coding и настройка «Кокпита»
- Теория (30 мин): Что такое Vibe-coding? Почему теперь не нужно учить синтаксис, а нужно учить логику. Обзор инструментов: почему терминал — это не страшно.
- Практика (90 мин): * Установка VS Code и расширений.
- Настройка Claude Code и Antigravity.
- Создание первой папки проекта.
- Команда: «Сделай мне простую веб-страницу с моим резюме и кнопкой смены темы».
- Результат: Работающая локальная страница и настроенная среда.

#### Занятие 02: Генерация интерфейсов «словами»
- Теория (30 мин): Как описывать идеи, чтобы ИИ понимал структуру. Разница между «сделай красиво» и «используй современный минимализм и темную тему».
- Практика (90 мин): * Работа в Bolt.new или Lovable.
- Создание структуры будущего продукта (интернет-магазин, дашборд или игра).
- Экспорт кода в свою рабочую среду для дальнейшей доработки агентами.
- Результат: Готовый визуальный прототип проекта в браузере.

#### Занятие 03: Claude Code — ИИ-руки в вашем компьютере
- Теория (30 мин): Возможности CLI-агентов. Как агент может сам создавать файлы, папки и устанавливать зависимости (библиотеки) без вашего участия.
- Практика (90 мин): * Даем Claude Code задачу: «Создай систему логики для моего проекта».
- Наблюдение за тем, как ИИ пишет код и исправляет свои ошибки в реальном времени.
- Отработка навыка «диалога с кодом»: как просить изменить конкретную деталь.
- Результат: Проект обрастает функционалом (кнопки начинают работать, данные сохраняются).

---

### Модуль 2: Профессиональные навыки и Экономика

#### Занятие 04: Экономика токенов и jcodemunch
* **Теория:** Как работают контекстные окна и почему «кормить» ИИ всем проектом — это дорого.
* **Практика:** Использование **jcodemunch** для сборки компактного контекста. Настройка `.claudignore`.
* **Итог:** Снижение затрат на API в 5–10 раз без потери качества.

#### Занятие 05: Скиллы для агентов и MCP
* **Теория:** Model Context Protocol (MCP) — как дать ИИ доступ к Google, базам данных и вашим файлам.
* **Практика:** Подключение внешних инструментов. Обучение агента работать с реальными данными из интернета.
* **Итог:** Агент, который умеет искать информацию и анализировать ваши документы.

#### Занятие 06: Оркестрация и агентные команды
* **Теория:** Агентная инженерия. Роли: «Архитектор», «Исполнитель», «Тестировщик».
* **Практика:** Запуск **Antigravity** для координации нескольких ИИ. Автоматическая проверка кода на ошибки.
* **Итог:** Работающая «мини-студия» из ИИ-агентов на вашем компьютере.

---

### Модуль 3: Инфраструктура и Свобода

#### Занятие 07: Свой сервер и Open Source (Infinite Tokens)
* **Теория:** Когда API становится дорогим. Обзор RunPod и Vast.ai.
* **Практика:** Аренда GPU-сервера за $0.30/час. Развертывание **DeepSeek-V3** или **Llama-3**.
* **Итог:** Личный безлимитный ИИ-кодер на удаленном сервере.

#### Занятие 08: ИИ как Системный администратор
* **Теория:** Управление серверами через SSH без знания команд Linux.
* **Практика:** Команда агенту: «Зайди на мой сервер, установи Docker и запусти мой проект».
* **Итог:** Проект опубликован в интернете и доступен по внешней ссылке.

---

### Модуль 4: Дипломная работа

#### Занятие 09: Большая сборка (Интенсив)
* Финальная сборка выбранного продукта (игра, ИИ-агент для валидатора, магазин).
* Устранение логических тупиков под присмотром ментора.

#### Занятие 10: Защита и Финализация
* Исправление багов через «самодиагностику» агентов.
* Демонстрация готового продукта.
* **Итог:** Студент уходит с работающим приложением и навыками для создания любого IT-проекта в будущем.

## Дипломная работа
Условие окончания школы — **полностью рабочий продукт**, созданный самостоятельно при помощи ИИ-агентов. 
> **Важно:** Ментор сопровождает вас до момента успешной сдачи. Если проект не работает — ментор проводит дополнительные консультации до победного финала.

## Зачем проходить обучение в ИИ-Школе?
1.  **0% скучной теории:** Мы не учим синтаксис Python или JavaScript.
2.  **100% контроля:** Вы учитесь управлять инструментами, которые пишут код за вас.
3.  **Эффективность:** Вы узнаете, как тратить копейки на токены там, где другие тратят тысячи.
4.  **Обучение по календарю и в группе:** На вас ложится дополнительная ответсвенность, которая не даст вам прокрастинировать.
5.  **Реальный результат:** Обучение не закончится, пока вы не создадите реальный продукт, который сможете продать или использовать сами.
